# features/workspace

Workspace management UI — sidebar listing, directory picker, session grouping.
Connects to main-process IPC for workspace CRUD, session listing, and native open actions.
Also owns the sidebar interaction contract between workspace groups and the launcher route.
Expanded Workspace sections use Session as the single sidebar list entity. Primary
Work Sessions participate in the same ordering, preview limit, and Session Group
partitioning as ordinary Chats, with Work metadata decorating their rows and routing
them to the Work surface.

## Files

- **index.ts**: Barrel re-exports for the workspace feature
- **file-tree-loader.ts**: Workspace file tree 的共享 lazy loader 与 intent preload 入口，供 right aside Files tab 使用
- **file-tree.tsx**: Right-aside workspace file tree using `@pierre/trees`, with persistent model-backed i18n-backed search, Git status annotations sourced from the shared Git status hook, active refresh for external file changes, double-click BrowserPanel tabs that open text/code files in the editor and rich files in preview, Space-triggered BrowserPanel preview tabs, shared workspace-owned Base UI file context menu actions, workspace file drag payloads for chat/TUI drops, and a first-render performance completion mark once workspace files and Git status are ready.
- **workspace-file-menu.tsx**: Shared workspace-owned file menu and file-operation dialog helpers reused by Files and Changes trees; owns BrowserPanel open, default-app open, Finder reveal, create file/folder, inline rename API wiring, VS Code-style relative/absolute path copy shortcuts, and Base UI collision-aware menu rendering.
- **workspace-file-preview.tsx**: Embedded read-only rich workspace file preview content for BrowserPanel tabs; uses workspace preview metadata to render Markdown with Streamdown, code previews with Shiki, browser-native images, PDF previews, Office-to-PDF renditions, and unsupported file fallbacks. Enter/double-click opens the text editor only for text and Markdown files.
- **workspace-pdf-preview.tsx**: PDF.js canvas renderer used for native PDF files and server-generated Office PDF renditions.
- **workspace-file-editor.tsx**: Monaco-backed editable workspace file surface for workspace-relative files, with language mapping, line numbers, folding, dirty-state feedback, save controls, and Cmd/Ctrl+S write-through to the workspace file content API.
- **workspace-file-language.ts**: Shared workspace file language helpers mapping file names/extensions to Monaco and Shiki language ids.
- **use-workspace-file-content.ts**: Hooks and URL builders for workspace file text content, preview metadata, raw bytes, and PDF renditions through workspace-owned APIs.
- **use-workspace-files.ts**: Workspace file search helper/hook for composer mentions, smart mentions, and quick open, using the bounded workspace file search API instead of preloading recursive inventories.
- **use-workspace.ts**: Hooks for listing, adding (via native directory picker), pin toggling, and deleting workspaces; exposes list readiness for interaction-level performance gates that depend on workspace names.
- **use-session.ts**: Hooks for listing active or archived sessions globally or under a single workspace, with memoized row normalization, session layout metadata hydration for app chrome, latest-user-message activity timestamps for list display, and targeted session-list cache updates; ordinary row patches preserve the API's latest-user-message order and existing timestamps, while explicit optimistic promotion is reserved for new user messages or newly created chat sessions.
- **use-cli-agents.ts**: Transitional hook for listing CLI-TUI Agent Profiles from the unified Agent Runtime
- **workspace-sidebar-ui-store.ts**: Workspace-owned persisted UI state for sidebar project group collapse and per-project full session-list expansion, using the shared safe localStorage wrapper and bounded workspace-id flag maps.
- **preview-card/**: Sidebar preview-card lifecycle and type-specific renderers. The shared provider keeps the interaction warm while crossing rows without previews, and `session/` owns transcript-backed session preview content.
- **workspace-sidebar.tsx**: Sidebar component whose workspace name opens the detail route surface for that workspace while the folder icon controls collapse state; app-level navigation opens New Chat, Search, Automation, Usage, and Settings before workspace-owned sections; workspace groups support pinned-first project ordering with shared visible-menu and right-click context-menu actions for pin/unpin, native Finder opening, create file/folder, rename, path copy, and delete; sidebar-owned session rendering uses one global sessions query grouped by workspace before handing rows to each workspace group, avoiding per-workspace list requests; collapsed sidebar mode keeps only the top app navigation visible and suppresses workspace/project rows; workspace/session menus handle new-surface open, Electron tear-off window open, rename, server-backed read-state toggle, pin, Markdown copy, archive, and native Finder opening through the workspace i18n namespace; archiving a session closes matching chat route surfaces and Electron tear-off windows after the session-owned archive write succeeds; session rows can be dragged outside the Electron window or double-clicked to tear chat sessions into separate windows while sharing the chat session tear-off reservation used by header surfaces; session hover/focus/open calls the chat-owned session prefetch boundary before navigation without owning transcript query details; the deferred Kanban sidebar records its own boards-query-backed first-render gate; session rows expose their actions through right-click and the keyboard Context Menu shortcut without reserving a trailing overflow-button slot; session rows keep a complete `min-w-0` shrink chain so long titles truncate inside the app sidebar; workspace session groups expand/collapse without height animation and show a bounded preview that automatically expands to include every pinned and running session, retaining just-finished running sessions until the user opens them; key navigation entries, including Search and Automation, expose stable `data-testid` anchors for E2E regression; unread display and manual read-state changes are driven by the server Session projection, and running/error display prefers the session API status with chat store as a local immediate supplement
