# Apps/Desktop/Scripts

Desktop-owned build and verification helper scripts.

## Files

- **build-mac-bridge.mjs**: Builds the Swift `cradle-mac-bridge` executable on macOS, atomically replaces the deterministic desktop packaging binary, and copies Mac Bridge runtime resources such as `Appshot.wav` into `.build/cradle-dist/resources`. Non-macOS hosts skip this step so shared CI can continue to build non-macOS slices.
- **generate-update-manifest.mjs**: Copies a macOS zip artifact into `release/update/macos`, calculates SHA-256 and size metadata, and writes the Cradle desktop update manifest consumed by the macOS self-update runtime.
- **record-appshot-parity.mjs**: Records observe-only Codex UI Appshot evidence and Cradle native Appshot evidence into `docs/manual-reports`. The script owns Appshot parity evidence collection only; it is unrelated to desktop packaging or updates.
- **run-update-smoke.mjs**: Prepares a local update feed from a new macOS zip artifact, serves it on `127.0.0.1`, and optionally launches an old packaged `.app` with `CRADLE_DESKTOP_UPDATE_URL` set for manual Desktop settings smoke.
- **set-version.mjs**: Updates `apps/desktop/package.json` to the release version used by Electron Builder and the Cradle desktop update manifest.
- **verify-macos-distribution-credentials.mjs**: Checks local macOS Developer ID Application and Developer ID Installer identities, optional notary profile access, and the active `electron-builder.mjs` signing configuration before starting expensive distribution packaging.

## Desktop Build

Desktop packaging is owned by `electron-builder.mjs`. The package scripts always pass that config explicitly:

    pnpm --filter @cradle/desktop pack
    pnpm --filter @cradle/desktop dist:mac
    pnpm --filter @cradle/desktop dist:publish

`electron-builder.mjs` consumes the server-owned `apps/server/dist/desktop-runtime` artifact as an extra resource. It does not install or select server dependencies; `@cradle/server` owns that runtime through `pnpm --filter @cradle/server build:desktop-runtime`.

macOS update zip artifacts are generated by Electron Builder. After `dist:mac`, generate the manifest upload directory:

    pnpm --filter @cradle/desktop generate-update-manifest -- \
      --base-url "https://updates.example.com/cradle"

Upload `apps/desktop/release/update` so the public update root contains `macos/manifest.json` and the referenced macOS zip. Set `CRADLE_DESKTOP_UPDATE_URL` to that public update root before building the app, or point it directly at the manifest JSON. Windows updates are generated by the NSIS Electron Builder target; publish the generated `latest.yml`, setup `.exe`, and `.blockmap` files in the same feed directory. The Windows runtime strips `manifest.json` from `CRADLE_DESKTOP_UPDATE_URL` and reads `latest.yml` beside it.

Rolling feeds use GitHub Actions run numbers for update ordering: `0.0.0-dev.<run-number>` for dev and `0.0.0-be.<run-number>` for bleeding edge. Commit hashes stay in release notes instead of the app version because SemVer prerelease ordering does not match commit chronology.

## Manual Update Smoke

自研 updater 的自动化测试覆盖 manifest 解析、下载、SHA-256 校验和 release manifest 生成。真正替换 `.app` 的路径仍需要 packaged macOS smoke，因为它依赖运行中 app 退出、目标目录权限、macOS administrator prompt 和 `open -n` relaunch。

推荐 smoke 流程：

1. 构建并安装一个旧版本 app。旧版本构建时设置 `CRADLE_DESKTOP_UPDATE_URL` 指向本地或公网 update root。
2. 构建一个新版本 macOS zip，并运行 `generate-update-manifest` 生成 `release/update/macos/manifest.json`。
3. 用静态文件服务暴露 `release/update`，确保旧版本 app 能读取 `<update-root>/macos/manifest.json` 和 manifest 中的 zip URL。
4. 在旧版本 app 的 Desktop settings 中执行 Check、Download、Restart。
5. 如果旧版本 app 安装在 `/Applications`，确认 macOS administrator prompt 出现并授权。
6. 等待旧版本主进程退出、installer script 替换 `.app`、新版本通过 `open -n` 启动。
7. 验证 About/Desktop settings 中版本号为新版本，并检查 `Application Support/Cradle/updates/last-update-result.json` 中 `ok` 为 `true`。

For a local feed and launch helper:

    pnpm --filter @cradle/desktop smoke:update -- \
      --old-app "/Applications/Cradle.app" \
      --new-zip "release/Cradle-0.0.2-universal.zip" \
      --version "0.0.2" \
      --launch

Keep the smoke helper running until the app finishes downloading the update. It serves the generated feed from a temporary directory and injects the update URL through the old app process environment.

## macOS Credentials

Run this preflight on a release machine before a signed macOS distribution build:

    pnpm --filter @cradle/desktop verify:macos-distribution-credentials -- \
      --mac-app-sign "Developer ID Application: Example Team (TEAMID)" \
      --mac-installer-sign "Developer ID Installer: Example Team (TEAMID)" \
      --mac-notary-profile cradle

The preflight is read-only. It does not import certificates, modify keychains, sign artifacts, notarize artifacts, install packages, or touch `/Applications`.
