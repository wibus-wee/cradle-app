> Cradle 正式开源！分屏工作区让你同时并排查看多个会话，插件生态正式起步，Workspace 间可以自由迁移 Issue 和看板，OpenCode 从每会话独立进程改为全局共享服务大幅降低资源占用，Diff Review 引导体验重写，以及一轮从流式架构到桌面端安装包的全面加固。

## ✨ 新功能

- **Cradle 正式开源！** 完整代码库已公开，附带全新 Landing Page。欢迎 Star & PR 🎉
- **分屏工作区** — 基于 Dockview 实现 Chat Session 分屏，从侧边栏拖拽会话即可创建分屏，支持多面板并排、聚焦联动和自定义布局
- **插件源管理** — Settings 新增插件来源管理界面，支持添加外部插件源；新增 Plugin Source 注册表（DB + API + CLI 全链路）
- **Codex++ 插件** — 首个外部 Provider Source 插件（由 @AsahiShiori 贡献 #7），可自动读取 Codex Relay 配置并将 ChatGPT 凭证映射为 Cradle Provider，一个命令搞定迁移
- **Workspace 迁移** — 右键 Workspace 即可发起迁移向导：选择目标 Workspace → 映射状态和里程碑 → 预览变更 → 一键执行。支持 Issue、看板、自动化定义的批量迁移
- **Worktree 管理与 Session 隔离** — 新增 Worktree 创建、清理策略设置（最大数量 / 最大占用空间），Session 级隔离让你的并行任务互不干扰
- **Relay Host 注册** — 通过配对字符串将远端主机注册到本地控制器，支持网络入站访问模式（仅局域网 / 公网）及公共 URL 配置
- **Claude Agent 工具审批** — Claude Agent 执行过程中需要你确认工具调用时，系统托盘会弹出通知提醒，审批完成后继续执行
- **DevTool 追踪面板** — 新增 IPC 与 ACP 追踪面板，调试内部通信事件一目了然
- **实时同步网关** — WebSocket 连接实现跨端实时数据同步
- **外部终端应用偏好** — 在设置中指定你偏好的外部终端应用
- **Kanban 支持 Workspace** — 看板视图绑定到 Workspace 维度，不同项目互不干扰
- **Issues 绑定 Workspace** — Issue 服务支持 Workspace 维度关联
- **多语言更新** — 西班牙语、日语、中文翻译全面补齐，新增 Session 隔离及 Workspace 相关词条

## 💎 改进

- **OpenCode 架构重构** — 从每会话独立进程改为全局共享服务器，启动时预热一次、所有会话复用，内存占用大幅下降；Resource Panel 可实时查看 OpenCode Server 的 RSS / CPU / 运行时间
- **ChatStreamChunk 流式架构** — 引入 replay / live 标记与 Stream Lease 生命周期管理，流式响应更可靠
- **SSE 性能优化** — 新增 buffered chunk stream，初始连接延迟更低
- **Chat Runtime 事件溯源** — 重构为原生事件驱动架构，快照处理与合并逻辑更健壮
- **Claude Agent 任务生命周期** — 跟踪任务活跃状态，Subagent Crew 链接共享更顺畅
- **工具分类统一** — 引入 CradleToolKind 统一词汇表，服务端工具分类更准确
- **Diff Review 引导 UX 重写** — 快照保留与深度链接导航，引导体验焕然一新
- **Composer 工具栏** — 状态管理增强，交互更流畅
- **快捷键覆盖层** — hold 检测逻辑改进，长按提示更准确
- **Accordion 重写** — 分组 proximity hover + ThinkingIndicator，思考过程展示更自然
- **子进程管理统一** — Runtime 统一管理子进程关闭流程，退出更干净
- **Remote Hosts 设置** — 面板重构，新增配对向导引导连接
- **认证与安全加固** — 本地信任边界收紧，路由与持久化状态健壮性提升
